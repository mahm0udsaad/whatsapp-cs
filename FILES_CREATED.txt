API ENDPOINT FILES (src/app/api/)
================================================================================

5. src/app/api/webhooks/twilio/route.ts (414 lines)
   PURPOSE: Main WhatsApp message webhook
   METHOD: POST
   FLOW:
     1. Parse Twilio form data
     2. Find restaurant by phone number
     3. Find or create conversation
     4. Save customer message
     5. Load AI agent config
     6. Query knowledge base for RAG
     7. Load menu items for context
     8. Generate AI response
     9. Save AI message
     10. Send via Twilio
     11. Return TwiML response
   DEPENDENCIES: types, gemini, twilio, admin
   ERROR HANDLING: Comprehensive with fallbacks
   RESPONSE: TwiML XML
   TIMEOUT: < 15 seconds (Twilio requirement)

6. src/app/api/webhooks/twilio/status/route.ts (98 lines)
   PURPOSE: Track message delivery status
   METHOD: POST
   HANDLES:
     - sent: Message sent from Twilio
     - delivered: Message delivered to WhatsApp
     - read: Customer read the message
     - failed: Message delivery failed
   DEPENDENCIES: types, admin
   RESPONSE: JSON { success: true }
   UPDATES: campaign_recipients table

7. src/app/api/menu/crawl/route.ts (367 lines)
   PURPOSE: Crawl restaurant websites for menu items
   METHOD: POST
   INPUT: { restaurant_id: string, url: string }
   FLOW:
     1. Fetch HTML from URL (with timeout)
     2. Parse HTML with Cheerio
     3. Try multiple CSS selector patterns
     4. Fallback to table parsing
     5. Extract: name, price, category, image
     6. Detect currency (SAR, AED, $, €, £)
     7. Save menu items to database
     8. Create knowledge base entries for RAG
     9. Return sample of extracted items
   DEPENDENCIES: cheerio, types, admin
   RESPONSE: JSON with items_extracted, items[], knowledge_base_entries
   ERROR HANDLING: Graceful with detailed error messages
   TIMEOUT: 30 seconds
